## 📋 Вариант 1: С Web App (требует HTTPS)

### Шаг 1: Установите ngrok для локального тестирования

```bash
# Установка ngrok (macOS)
brew install ngrok

# Или скачайте с https://ngrok.com/download
```

### Шаг 2: Запустите Web App сервер

```bash
# В отдельном терминале
cd /Users/kristina/kris_/bot_tg
python3 webapp_server.py
```

Сервер запустится на порту 8081.

### Шаг 3: Создайте HTTPS туннель через ngrok

```bash
# В другом терминале
ngrok http 8081
```

Вы получите URL типа: `https://abc123.ngrok.io`

### Шаг 4: Обновите URL в handlers.py

Откройте `handlers.py` и найдите строку (примерно 238):

```python
webapp_url = f"https://your-ngrok-url.ngrok.io/webapp/channels?channels={channels_encoded}"
```

Замените `your-ngrok-url.ngrok.io` на ваш ngrok URL:

```python
webapp_url = f"https://abc123.ngrok.io/webapp/channels?channels={channels_encoded}"
```

### Шаг 5: Перезапустите бота

```bash
ssh root@144.31.221.168 'systemctl restart tg-bot'
```

## 🆘 Troubleshooting

### Ошибка "Web App URL must use HTTPS"

Telegram требует HTTPS для Web App. Используйте ngrok или разместите файл на сервере с SSL.

### Не работает кнопка Web App

1. Проверьте, что сервер запущен на порту 8081
2. Проверьте, что ngrok туннель активен
3. Убедитесь, что URL в handlers.py правильный
4. Проверьте логи бота: `ssh root@144.31.221.168 'journalctl -u tg-bot -f'`

### Web App открывается, но каналы не загружаются

1. Проверьте консоль браузера (F12)
2. Убедитесь, что URL содержит параметр `?channels=...`
3. Проверьте, что в базе есть каналы с заполненными title
