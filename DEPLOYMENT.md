# 🚀 Инструкция развертывания бота на сервере

## Что было добавлено в последнем обновлении

### 1. **Балансировка между 3 аккаунтами при FloodWait**
- Основной аккаунт: `ltdnt_session`
- Резервный аккаунт: `211766470_telethon`
- Третий аккаунт: `kristina_user`

### 2. **Защита от FloodWait**
- Минимум 3 минуты между запросами пользователя (было 2)
- 5-секундная пауза после успешного анализа
- Только 1 анализ одновременно (очередь)

### 3. **Отслеживание незавершённых анализов**
- Новая таблица `pending_analyses` в БД
- Автоматическое сохранение анализа при FloodWait
- Админ-команда `/send_pending` для отправки уведомлений платящим юзерам

## Как развернуть на сервере

### На сервере:

```bash
cd /opt/bot_tg

# Обновить код из git
git pull origin 2026-01-17

# Обновить зависимости если нужны новые
pip install -r requirements.txt

# Перезапустить сервис
sudo systemctl restart tg-bot
```

### Проверить статус:

```bash
# Посмотреть логи
sudo journalctl -u tg-bot -f

# Проверить работает ли
curl http://localhost:8080/health
```

## Новые команды для администратора

### `/send_pending`
Отправляет уведомления платящим пользователям о их незавершённых анализах.

Когда использовать: после восстановления бота после сбоя при FloodWait.

```
/send_pending
```

## Что происходит в процессе

1. **Пользователь запрашивает анализ** → Бот пробует основной аккаунт
2. **FloodWait на основном** → Автоматически переключается на резервный
3. **FloodWait на резервном** → Использует третий аккаунт
4. **Все 3 в FloodWait** → Анализ сохраняется в `pending_analyses`, отправляется уведомление
5. **Админ выполняет `/send_pending`** → Юзеры получают список для переанализа
6. **Юзер повторно отправляет** → Анализ успешно обрабатывается

## Возможные проблемы и решения

### "Conflict: terminated by other getUpdates request"
- На сервере уже запущен бот
- Решение: `sudo systemctl restart tg-bot`

### "FloodWait" частый
- Слишком много запросов подряд
- Решение: увеличить `RATE_LIMIT_SECONDS` в handlers.py

### БД заполняется флудвейтами
- Решение: `/clear_floodwait_db` - удалить все записи
